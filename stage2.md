# 阶段 2

我只需要弄清楚事情是如何运作的。——伊万·萨瑟兰

> 伊万·萨瑟兰（Ivan Sutherland, 1938 - 今），计算机图形学之父，1963 年开发了首个交互式图形系统 Sketchpad，于 1972 年转至犹他大学任教，并在那里培养了一批后来奠定现代计算机图形学和电子游戏产业基础的学生。
---

我们已经领略了很多图形学的侧面，希望它正如你想象的那样有趣或令人惊叹，如果超乎你的想象就再好不过了！

在这个阶段，我们希望你深入到真正的计算机图形学，触摸她的灵魂——而不是仅仅关注到*外表*。

> *外表*：指各种具体的引擎工具、图形 API 等。

这就好像计算机科学和编程语言的关系，如果仅停留在后者而忽略前者，那么你就会沦为 **Code Monkey** 而不是成为 **Craftsman** ——尤其是在 Agentic Coding 越发强悍的时代，对基础的理解已经比以往任何一个时代都更重要。

你仍然需要进行大量图形编程练习，但它们不再是学习的唯一重心——你会投入比以往更多的精力去建立各种系统、算法和技术的心智模型。

好了，多说无益，来看看你即将开启的新旅程吧！

## 正式入门

我们让大家观看犹他大学（没错，就是开头人物介绍那个犹他大学）的 *CS 4600 - Introduction to Computer Graphics* 课程，这是综合考虑下来最适合我们 TDSG 的图形学入门课程，由 Cem Yuksel 授课，他讲课的风格很有趣，人也特别有趣，目前正饱受 YouTube 评论区无脑原教旨光栅化渲染孝子的困扰哈哈哈哈哈(/≧▽≦)/

完成图形学入门后，你应该能够回答下面这些基础问题，这些问题将同时是结束课程章节后的 quizzes：

### 图像是怎么来的
- 一张图片是如何存储在计算机里的？
- 一个三维场景怎么变成二维画面？
- 一个像素最终是如何被计算出来的？

### 模型是怎么动起来的
- 一个模型怎么旋转、缩放和移动？
- 摄像机如何观察场景？
- 动画为什么能够连续播放？

### 光照是怎么工作的
- 为什么同一个模型在不同光照下看起来完全不同？
- 阴影是怎么实现的？
- 反射和折射是怎么实现的？
- PBR 好在哪里？
- 为什么说一切工作都是在逼近渲染方程？

### GPU 是怎么工作的
- Shader 到底是什么？
- GPU 为什么比 CPU 更适合渲染？

### 图像为什么会有各种问题
- 为什么会出现锯齿？
- 为什么远处会闪烁？
- 为什么需要纹理过滤？

其实你现在就可以检验一下目前对图形学基础的了解有多少。不过光有这些理解还不够，还需要亲手做几个 project 才行。
